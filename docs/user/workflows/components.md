# 组件管理

组件库用于沉淀工作空间级 Vue 组件，让项目页面可以复用稳定版式、交互和视觉规范。

## 组件基本信息

组件应包含清晰的名称、描述、适用场景和参数说明。名称建议体现业务用途，例如“封面标题区”“三列指标卡”“时间线步骤页”。

## 草稿与发布版本

组件采用草稿与发布版本模型。草稿用于编辑和调试，发布版本用于页面稳定引用。破坏性调整应发布新版本，并让页面按需升级，不要直接改变已被大量页面依赖的行为。

## 组件工作台

组件工作台用于编辑源码、配置预览 schema、查看预览效果和发布版本。预览 schema 应覆盖组件的典型参数组合，方便设计、开发和 AI 判断组件能力边界。

## 离线分享包

离线分享包适合跨环境迁移组件能力。导出前应确认组件依赖的资源、字体、主题和 Runtime Kit 路径都在目标环境可用。

## 与 Runtime Kit 的关系

组件源码只能引用 Runtime Kit manifest 中公开且版本化的能力，例如 `<ExportName>.v1` 对应的公开 import path。未进入 manifest 的 Runtime shell 内部能力不应被组件引用。

## 与资源和字体的关系

组件可以引用工作空间资源和字体注册。组件发布前应检查依赖是否完整，避免在其他项目中预览或构建失败。

## 与 AI 协作

AI 适合辅助生成组件初稿、补充预览 schema、解释参数约束和改写局部样式。发布前仍需要人工确认源码、依赖和预览效果。

## 使用建议

- 高频页面结构优先抽成组件。
- 组件参数保持清晰，避免把页面特有内容写死。
- 破坏性变更用新版本承接。
- 发布版本前至少用典型 schema 做一次预览验证。
