# 主题、字体与样式

主题、字体和样式共同决定项目视觉约束。它们不是单页装饰，而是面向团队复用的设计系统资产。

## 当前对象边界

- 主题：维护品牌色、Logo、页面图标、默认字体绑定和项目视觉基础配置。
- 字体注册：把字体资源转为页面和组件可引用的字体声明。
- 样式库：沉淀页面展示配置、Markdown 样式规范和通用排版约束。

## 主题库

主题适合保存组织级或项目级视觉规范。创建主题时，应明确适用场景、主色、辅助色、背景策略、Logo 资源和默认字体。

## 字体注册

字体注册依赖字体资源。上传字体后，需要完成字体声明配置，页面和组件才能稳定引用。生产部署时还要确认字体资源可以被 Runtime 正确加载。

## 样式库

样式库适合保存不同内容形态的排版约束，例如报告页、图文卡片、数据看板、演讲页。样式说明应面向人和 AI 都可理解，避免只写抽象口号。

## 离线包

主题、字体、样式和组件可能共同构成一个可迁移的设计系统包。迁移前应检查关联资源、字体文件和组件依赖，避免目标环境缺失素材。

## 推荐使用流程

1. 先上传品牌素材和字体资源。
2. 注册字体并创建主题。
3. 补充样式库说明。
4. 创建或发布与主题匹配的组件。
5. 在项目中应用主题和样式，再开始页面创作。

## 与 AI 协作

AI 会更容易遵循明确的主题、字体和样式约束。让 AI 改版前，应说明目标主题、页面比例、参考组件和禁用风格。

## 注意事项

- 不要把临时页面样式沉淀为全局样式。
- 字体资源变更会影响所有引用页面，替换前需要检查范围。
- 样式说明应具体到布局、密度、色彩和可读性要求。
