# 项目与页面

项目与页面是平台创作流程的主线。项目承载页面集合、主题样式、资源引用和构建配置；页面承载具体内容源码和预览结果。

## 创建项目

进入工作空间后创建项目，先确认页面比例、主题、默认样式和项目说明。项目说明会影响团队协作理解，也会在 AI 上下文中帮助模型判断内容目标。

## 组织页面

页面建议按最终演示顺序命名，并在标题中体现主题，例如“01 封面”“02 核心结论”“03 数据拆解”。页面数量较多时，应把稳定结构沉淀为组件，避免每页复制相同代码。

## 编辑页面源码

页面源码应优先使用平台允许的 Runtime Kit 公开能力、工作空间组件和资源引用。不要依赖 Runtime shell 内部组件，也不要引用未版本化的 `@runtime-kit` 路径。

## 保存与版本

保存页面会写入 Backend。需要回看历史内容时，使用页面版本能力定位变更；需要跨项目复用时，优先把稳定片段抽成组件或样式，而不是复制页面源码。

## 预览与构建

编辑后通过 Editor 预览区查看 Runtime 渲染效果；需要交付时发起项目构建，由 Backend 创建构建任务，Runtime 生成构建产物并回传。完整流程见 [预览、截图与构建](./preview-build-export.md)。

## 与 AI 协作

适合交给 AI 的任务包括页面结构初稿、局部文案改写、版式优化、组件调用示例和样式微调。涉及删除页面、覆盖源码或批量替换时，应认真检查工具确认内容。
