# 预览、截图与构建

预览、截图和构建由 Backend、Editor 和 Runtime 协同完成。用户通常只需要在 Editor 中保存、预览和发起构建，但理解链路有助于排查问题。

## 页面预览

页面保存后，Backend 创建预览上下文并签发短期令牌，Runtime 通过预览入口加载页面源码、配置包、资源和组件，Editor 使用 iframe 展示渲染结果。

## 组件预览

组件预览依赖组件源码和 previewSchema。previewSchema 应包含典型输入，方便确认组件在不同参数下是否稳定渲染。

## 截图

截图能力用于生成页面缩略图、预览图和文档展示图。截图失败通常与页面运行错误、资源加载失败或 Runtime 服务不可用有关。

## 项目构建

发起构建后，Backend 创建构建任务和 build snapshot，Runtime 拉取 snapshot 并执行 Vite 构建，再把产物回传给 Backend。构建成功后，用户可以通过稳定链接访问或下载产物。

## 常见问题

- 预览空白：先检查页面源码是否报错，再检查资源和组件引用是否存在。
- 图片或字体缺失：检查资源是否归档、删除或未被 Runtime 正确访问。
- 构建失败：查看构建任务状态和 Runtime 日志，确认源码、依赖和 manifest 引用。
- 截图超时：确认 Runtime 服务可用，页面没有长时间阻塞渲染。
