# 资产复用

平台的核心价值之一是把演示文稿相关素材沉淀为可复用资产。资源、组件、主题、字体和样式进入同一个工作空间后，创作者和 AI 都可以基于稳定上下文继续创作。

## 资源库

资源库用于保存图片、图标、视频、字体、DrawIO、Mermaid、图表、公式和可编辑内容资源。相比把素材散落在页面代码里，资源库能让团队统一命名、替换、归档和追踪引用关系。

适合沉淀为资源的内容包括：

- 品牌 Logo、产品截图、人物照片和背景图。
- 常用图标、图表配置、公式和 Mermaid 图。
- 字体文件和可复用视觉素材。
- 后续可能需要 AI 继续改写的结构化内容。

操作流程见 [资源管理](../workflows/resources.md)。

## 组件库

组件库用于沉淀工作空间级 Vue 组件，适合复用稳定版式、交互和视觉结构。例如封面标题区、三列指标卡、时间线步骤页、报告摘要页和数据图表区块。

组件采用草稿与发布版本模型：

- 草稿用于编辑、调试和补充预览输入。
- 发布版本用于页面稳定引用。
- 破坏性调整应发布新版本，再让页面按需升级。

组件使用流程见 [组件管理](../workflows/components.md)。

## 主题、字体与样式

主题、字体和样式共同约束项目视觉方向：

- 主题保存品牌色、Logo、项目图标和默认字体绑定。
- 字体注册把字体资源转为页面和组件可引用的声明。
- 样式库沉淀页面尺寸、基础字号、图标描边、菜单模式和 Markdown 样式规范。

这些资产能帮助 AI 按团队风格生成页面，而不是每次重新猜测视觉规范。使用流程见 [主题、字体与样式](../workflows/design-system.md)。

## 推荐沉淀顺序

1. 上传品牌素材、字体和常用图片。
2. 注册字体并创建主题。
3. 补充样式库说明，让人和 AI 都能理解排版边界。
4. 把高频页面结构做成组件并发布版本。
5. 在项目中应用主题、样式和组件，再批量扩展页面。

## 与 AI 协作

AI 更容易使用结构清晰、描述完整的资产。资源和组件应补充名称、说明、适用场景和关键参数；主题和样式说明应具体到布局、密度、色彩和可读性要求。

如果资产命名模糊，AI 往往需要猜测用途。建议用“季度收入柱状图 v1”“品牌 Logo 主色版”“三列指标卡组件”这类能直接表达内容和用途的名称。
