# AI 协作创作指南

平台 AI Agent 会读取当前用户、工作空间、项目、页面和工具说明等上下文，帮助完成内容生成、页面改写、组件管理和资源使用建议。AI 更适合承担结构化、可描述、可验证的创作任务。

![AI 工作台占位图](../assets/ai-workbench-placeholder.svg)

## 适合交给 AI 的任务

- 生成页面内容结构，例如章节页、报告摘要、数据结论和行动建议。
- 调整页面视觉方向，例如更换布局、统一标题层级、优化信息密度。
- 根据现有页面创建相近风格的新页面。
- 生成或修改工作空间组件，让多页复用相同模块。
- 根据资源库素材提出页面引用建议。

## 提示词建议

向 AI 描述任务时，优先包含四类信息：

| 信息 | 示例 |
| :--- | :--- |
| 目标 | “把当前页改成产品发布会开场页” |
| 范围 | “只修改正文区，不要改项目主题” |
| 约束 | “保持 16:9，文字不超过三层层级” |
| 验收 | “预览时标题、三条卖点和底部日期都要清晰可见” |

## 工具确认

AI 执行某些工具前会进入确认状态。用户需要检查工具名称、参数、影响范围和风险说明，再决定是否继续。常见需要确认的场景包括：

- 写入或覆盖页面源码。
- 创建、修改或删除组件。
- 批量调整资源、主题或样式。
- 触发可能影响构建结果的操作。

## 上下文边界

AI 只应基于当前用户有权限访问的工作空间、项目、页面和资源工作。没有进入上下文的外部文件、私有链接或未上传素材，AI 默认不可见。需要 AI 使用某个素材时，优先先上传到资源库，再在对话中说明素材名称和用途。

## 协作节奏

- 大改动先让 AI 给出方案或创建草稿，再确认具体落地。
- 页面视觉问题用预览结果描述，例如“右侧文字拥挤”或“封面主标题不够突出”。
- 对关键页面保留明确目标，避免一次请求同时修改内容、布局、主题和资源引用。
- 当 AI 提出工具确认时，先核对作用对象是否是当前页面或预期组件。
