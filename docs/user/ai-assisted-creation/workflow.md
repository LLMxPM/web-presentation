# 协作流程与提示词

## 适合交给 AI 的任务

- 生成页面内容结构，例如章节页、报告摘要、数据结论和行动建议。
- 调整页面视觉方向，例如更换布局、统一标题层级、优化信息密度。
- 根据现有页面创建相近风格的新页面。
- 生成或修改工作空间组件，让多页复用相同模块。
- 根据资源库素材提出页面引用建议。
- 维护项目页面元数据、项目路由和项目样式配置。

## 提示词建议

向 AI 描述任务时，优先包含四类信息：

| 信息 | 示例 |
| :--- | :--- |
| 目标 | “把当前页改成产品发布会开场页” |
| 范围 | “只修改正文区，不要改项目主题” |
| 约束 | “保持 16:9，文字不超过三层层级” |
| 验收 | “预览时标题、三条卖点和底部日期都要清晰可见” |

## 协作节奏

- 大改动先让 AI 给出方案或创建草稿，再确认具体落地。
- 页面视觉问题用预览结果描述，不只说“重新美化”。
- 对关键页面保留明确目标，避免一次请求同时修改内容、布局、主题和资源引用。
- 当 AI 提出工具确认时，先核对作用对象是否是当前页面、当前项目或预期组件。
- 如果 AI 的回答明显偏离当前上下文，先指出目标范围，例如“只处理当前项目，不要考虑其他工作空间资源”。

## 请求示例

```text
请把当前页面改成 16:9 的季度复盘封面。
保持当前项目主题，不新增图片资源。
主标题要突出，副标题和日期放在底部。
写入前先读取当前页面源码并说明将修改哪些区域。
```

```text
请根据当前项目的页面列表，新增一个“风险与应对”页面。
使用当前项目样式配置，页面内容控制在三列以内。
先创建可运行的页面，再把它加入项目路由。
```

```text
请检查资源库里是否已有适合做封面背景的资源。
如果没有合适的，只给建议，不要创建新资源。
```
