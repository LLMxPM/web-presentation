# 工具确认与会话

## 工具确认

AI 执行某些工具前会进入确认状态。用户需要检查工具名称、参数、影响范围和风险说明，再决定是否继续。

常见需要确认的场景包括：

- 写入或覆盖页面源码。
- 创建、修改、发布或删除组件。
- 写入资源内容、复制资源或归档资源。
- 更新项目样式配置、项目页面元数据或项目路由。
- 触发可能影响构建结果的操作。

确认前重点看四件事：

| 检查项 | 说明 |
| :--- | :--- |
| 目标对象 | 确认 `page_id`、`project_id`、`component_id`、资源名称或资源 ID 是否符合预期 |
| 操作类型 | 区分读取、预览、写入、发布、归档和删除 |
| 参数内容 | 页面 edits、组件 edits、资源文本内容、路由树或元数据是否符合请求 |
| 失败处理 | 如果工具返回校验失败或诊断信息，应让 AI 基于错误修正，不要绕过工具流程 |

## 会话范围

每个 AI 会话都会绑定创建时的业务范围。页面会话绑定具体页面，项目会话绑定具体项目，组件助手和资源助手通常绑定工作空间库入口。

切换页面或项目后，旧会话不会自动变成新页面会话。继续运行前，平台会检查当前路由是否仍在会话范围内；不一致时，用户应切回会话对应页面，或新建一个当前范围下的会话。

## 图片附件

AI 侧边栏支持上传图片附件供模型分析，但前提是当前绑定模型支持图片输入。当前前端允许上传 PNG、JPEG 和 WebP，单个附件大小上限为 10MB。

图片附件只作为本轮会话输入，不等同于工作空间资源。需要长期复用时，可以把会话图片附件提升为工作空间资源，再在页面或组件中引用。

## 中断和恢复

运行中可以停止当前任务。后端会以会话为单位记录 active run，刷新页面后可以恢复运行状态、待确认动作、上下文预算和最近事件。

如果一次运行已经进入工具确认或结构化提问状态，继续或拒绝都应在当前会话中完成。直接切换到另一个页面继续运行，可能会因为会话范围不匹配而被阻止。
