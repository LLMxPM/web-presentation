# 当前上下文注入

当前实现不是由 Editor 把所有页面、组件和资源一次性塞进对话消息，而是按“路由范围 + 后端校验 + 运行时上下文 + 工具按需读取”的方式工作。

## 注入流程

1. Editor 根据当前路由和当前选择的助手生成 `scope`。
2. Backend 校验当前用户是否能访问该工作空间，并校验页面、项目或组件是否属于该工作空间。
3. Backend 在需要运行或恢复 Agent 状态时，按 `scope` 构建 `AgentRuntimeContext`。
4. Agent 构建时把固定业务上下文追加到 Pydantic AI 的运行上下文中。
5. 工具调用时，Backend 会把当前 `scope`、画布尺寸、样式配置、工具授权令牌等作为工具依赖传入，工具再按权限读取或写入真实数据。

## 什么时候注入

| 场景 | 当前行为 |
| :--- | :--- |
| 打开 AI 侧边栏或切换助手 | Editor 发送当前 `scope` 查询可用助手；Backend 只做范围校验和可用性判断，不把完整业务上下文发给模型 |
| 创建会话 | 会话 metadata 会保存当时的 `scope`，包括工作空间、项目、页面或组件范围 |
| 发送新消息 | Backend 重新按当前 `scope` 构建运行时上下文，并随本轮模型运行注入 |
| 继续工具确认或结构化提问 | Backend 会优先使用 paused run 记录的 `run_scope` 重新构建上下文后继续运行 |
| 刷新页面、切换会话、查看上下文用量 | Backend 会构建运行时上下文，用于恢复会话状态、估算上下文预算和展示摘要 |
| 工具调用 | 工具通过本轮依赖拿到当前业务范围和授权令牌，再按工具自身规则读取或写入 |

运行时上下文按请求即时构建，通常会使用数据库里的最新页面、项目和组件信息。会话本身会保存创建时的工作范围；每次 run 也会记录自己的 `run_scope`。如果当前路由已经不在该会话范围内，平台会提示或阻止继续运行。

## scope 怎么来

| 入口 | 生成的范围 |
| :--- | :--- |
| 页面详情中的内容助手 | `page` 范围，包含 `workspace_id`、`project_id`、`page_id` 和来源 |
| 项目页面列表中的内容助手 | `project` 范围，包含 `workspace_id`、`project_id` 和来源 |
| 工作空间主页或无项目路由 | `workspace` 范围；内容助手可展示入口，但启动需要进入具体项目 |
| 组件助手 | `workspace` 范围，来源是组件库 |
| 资源助手 | `workspace` 范围，来源是资源库 |

前端还会同时维护“当前路由范围”和“当前会话范围”。两者不一致时，顶部状态会提示用户，运行前后端也会校验当前路由是否落在会话范围内。

## 自动注入哪些内容

固定业务上下文当前主要包括：

| 类别 | 内容 |
| :--- | :--- |
| 基础范围 | 范围类型、工作空间 ID、项目 ID、页面 ID、组件 ID、来源 |
| 项目展示配置 | 页面画布宽高、项目基础字号、基础字号和 Tailwind 尺度关系 |
| 项目样式规范 | 项目的 Markdown 样式规范文本，存在时注入 |
| 项目建议资源 | 建议优先参考的资源 ID、名称、原文件名、类型、描述、是否可编辑 |
| 页面元数据 | 页面标题、页面描述、页面编码、页面文件类型 |
| 组件元数据 | 组件编码、组件名称 |
| 工具可见性 | 内容助手会额外看到当前工具启用情况、模型是否支持图片输入，以及工具调用策略说明 |
| 会话历史 | 按模型上下文预算保留最近历史消息；超出预算时使用会话摘要和较少最近消息 |
| 用户输入 | 当前这一轮用户消息；如用户上传图片且模型支持图片输入，则包含图片附件 |

内容助手是 Team 入口。它的主助手和成员组件助手、资源助手会共享同一份业务范围；成员助手只在内容助手委派组件或资源任务时运行。

## 不会自动注入哪些内容

- 不会自动注入完整页面源码。AI 要精确修改页面时，应先通过页面读取工具获取当前源码。
- 不会自动注入项目页面列表、项目路由树、组件详情、组件源码、资源列表、资源内容或 Runtime Kit 详情。这些内容需要通过对应工具按需读取。
- 内容助手查询组件列表时，默认优先返回项目页维护的项目建议组件；如果当前项目没有建议组件或筛选后为空，会自动回退到当前工作空间已发布组件。需要完整组件库时，工具会显式按全库范围查询。
- 不会自动读取外部文件、私有链接或未上传素材。
- 不会自动读取页面截图；只有模型支持图片输入且截图工具可见时，AI 才能通过工具请求页面截图。
- 不会自动渲染资源效果。资源助手当前也不能预览资源渲染结果。

## 与上下文隔离的关系

当前隔离以工作空间成员权限、项目/页面/组件归属校验和会话范围校验为主。页面读取工具会校验目标页面是否属于当前工具上下文绑定的工作空间或项目；组件和资源工具也会按工具授权范围工作。

这是一种面向创作链路的软隔离：AI 可以在当前工作空间内按工具读取必要事实，但项目级上下文和工作空间级上下文仍在继续优化，后续会进一步降低当前页面任务、项目资产和工作空间资产之间的混淆概率。
