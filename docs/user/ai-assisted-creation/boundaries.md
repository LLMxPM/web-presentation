# 能力边界

## 内容助手

内容助手是页面和项目任务的主执行者，适合处理页面创建、页面改写、页面元数据维护、项目路由维护、项目样式配置读取与写入、页面检查和页面截图分析。

当前边界：

- 内容助手需要进入具体项目后才能启动。
- 页面源码不会自动进入固定上下文；修改前应通过工具读取页面源码。
- 组件和资源事实需要通过工具读取，不能凭空编造组件 import、资源名或 Runtime Kit 路径。
- 当任务需要创建、修改、发布组件或维护资源内容时，内容助手会按需调度组件助手或资源助手。

## 组件助手

组件助手面向工作空间组件库，适合处理组件草稿、组件源码 edits、组件元数据、组件发布、组件依赖和 Runtime Kit 能力查询。

当前边界：

- 组件归属于工作空间，不因为当前项目上下文改变归属。
- 组件源码和 preview schema 需要通过组件工具读取或写入。
- 组件只有发布后才能被页面按版本引用。
- 组件助手可以读取资源列表和可编辑资源内容，用于生成组件源码或 preview schema，但不负责页面写入。

## 资源助手

资源助手面向工作空间资源库，适合查询资源、读取可编辑资源内容、创建可编辑文本资源、维护元数据、复制和归档。

当前边界：

- 可直接操作的是 SVG 图标、DrawIO、Mermaid、图表配置、公式等可以文本编辑的资源。
- 当前不包括图片资源的内容生成或内容编辑。
- 暂时无法预览资源的渲染效果。
- 不暴露删除工具；用户要求删除时，当前只能执行归档或说明需要人工处理。
- 视频、字体、位图图片和位图图标只能由用户上传后维护元数据，不能由资源助手生成内容。

## 可见性边界

AI 只应基于当前用户有权限访问的工作空间、项目、页面、组件和资源工作。没有进入上下文、没有通过工具读取、没有上传到资源库或没有作为会话附件提供的内容，AI 默认不可见。

需要 AI 使用某个素材时，优先先上传到资源库，再在对话中说明素材名称和用途。需要 AI 精确比较或修改某段内容时，先让 AI 读取对应页面、组件或资源内容，再执行写入。
