# 当前上下文注入

上下文注入是平台让 AI 理解当前创作对象的机制。Backend 会根据用户所在位置和权限，把必要的项目、页面、组件、资源和样式信息组织给 Agent。

## 注入流程

1. 用户在 Editor 中打开项目、页面、组件或资源。
2. Editor 发起 AI 会话请求，并携带当前范围。
3. Backend 校验用户权限。
4. Backend 读取当前范围内的必要上下文。
5. Agent 根据上下文和用户提示词决定回复或工具调用。

## 什么时候注入

打开页面、切换会话、继续 run、发起工具调用前，都可能需要重新读取上下文。平台会尽量避免把无关对象塞进上下文，降低模型误判和成本。

## scope 怎么来

scope 通常来自当前 UI 位置，例如项目页、页面编辑页、组件工作台或资源详情。不同 scope 会影响 AI 能看到的对象和可调用工具。

## 自动注入哪些内容

- 当前用户和工作空间信息。
- 当前项目、页面或组件摘要。
- 可用资源、主题、样式和字体摘要。
- 已披露的工具说明、参数 schema 和返回示例。
- 当前会话的历史消息摘要。

## 不会自动注入哪些内容

- 用户无权访问的工作空间或项目。
- Runtime shell 内部实现细节。
- 未被平台披露的工具能力。
- 不必要的大体积文件全文。
- 密钥、凭证和生产环境敏感配置。

## 与上下文隔离的关系

上下文注入以权限和 scope 为边界。AI 不应绕过 Backend 直接读取其他用户、其他工作空间或 Runtime 内部文件。
