# 工具确认与会话

## 工具确认

AI 调用会影响平台数据的工具时，可能需要用户确认。确认前应重点检查：

- 操作对象是否正确。
- 参数是否符合预期。
- 是否会覆盖、删除或批量修改内容。
- 是否引用了正确的项目、页面、组件或资源。

低风险查询类工具通常可以直接运行；写入、删除、发布、构建等操作应谨慎确认。

## 会话范围

会话通常绑定当前工作空间、项目、页面、组件或资源。切换对象后，AI 看到的上下文会变化。需要跨对象操作时，应在提示词里明确说明来源和目标。

## 图片附件

用户上传图片后，平台会把图片作为会话上下文的一部分提供给 AI。涉及敏感内容时，不要上传到不可信模型供应商。

## 中断和恢复

会话或 run 中断后，平台会保留运行状态、事件、消息和工具调用记录。恢复前应先确认上一轮是否已经写入数据，避免重复执行相同操作。
