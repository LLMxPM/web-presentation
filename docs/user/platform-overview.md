# 平台介绍

`web-presentation` 是面向 AI 演示文稿创作的平台，用于创作 PPT、图文卡片、专题报告页、数据解读页等视觉化内容。平台把页面内容代码化，并把资源、组件、主题和样式沉淀为可复用资产，让团队可以在同一套工作空间中持续创作、预览、迭代和交付。

[平台总览图（DrawIO）](../assets/platform-overview.drawio)

## 适用场景

| 场景 | 适合产物 | 平台价值 |
| :--- | :--- | :--- |
| 演示文稿创作 | PPT 页面、演讲用页面、封面页 | 让 AI 参与结构生成、视觉改写和内容补全 |
| 报告页制作 | 数据解读页、专题报告、项目汇报页 | 复用团队主题、组件和资源，降低重复设计成本 |
| 图文内容生产 | 图文卡片、活动页、长图局部页面 | 使用 Runtime 快速预览视觉结果，按项目沉淀版本 |
| 团队资产管理 | 资源、组件、主题、样式 | 把素材和组件统一放进工作空间，支持跨项目复用 |

## 核心概念

- **工作空间**：团队资产和项目的隔离边界。资源、组件、主题和样式都归属于工作空间。
- **项目**：一组有共同目标的页面集合，例如一次产品发布、一份报告或一个专题内容包。
- **页面**：最终被预览、截图或构建的创作单元。页面内容以代码和结构化配置表达。
- **资源**：图标、图片、视频、字体、DrawIO、Mermaid、图表和公式等工作空间素材，供页面、组件、主题和字体注册引用，详见 [资源管理体系](./resource-management.md)。
- **组件**：可复用的 Vue 工作空间组件，采用草稿与发布版本模型，详见 [组件管理体系](./component-management.md)。
- **主题、字体与样式**：主题维护色板、Logo、项目图标和字体绑定；字体注册管理字体声明；样式库沉淀页面展示配置和 Markdown 样式规范，详见 [主题、字体与样式管理体系](./design-system-management.md)。
- **AI Agent**：理解当前工作空间、项目、页面和工具约束的协作助手。
- **Runtime**：负责在浏览器中渲染页面、组件预览、截图和构建产物的运行引擎。

## 平台组成

| 模块 | 用户看到的能力 | 说明 |
| :--- | :--- | :--- |
| Editor | 创作工作台 | 管理项目、页面、资源、组件、主题、样式和 AI 侧边栏 |
| Backend | 平台控制面 | 保存数据、校验权限、组织 AI 上下文、管理预览和构建任务 |
| Runtime | 预览与构建引擎 | 渲染页面、生成截图、执行诊断和构建 |
| 部署环境 | 私有化运行 | 通过平台镜像、Runtime 镜像和 compose 模板部署到自有环境 |

## 与其他创作路径的区别

传统 Keynote/PowerPoint、套模板的 AI PPT、Codex/Claude Code + PPT skills、OOXML/PPTX 生成、HTML 生成和图片生成路线都能解决不同阶段的演示文稿创作问题；`web-presentation` 更强调把页面、资源、组件、主题、样式和构建链路纳入平台模型，服务长期项目与团队资产沉淀。详细对比见 [演示文稿创作路径对比](./platform-comparison.md)。
