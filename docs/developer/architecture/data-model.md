# 核心数据模型

本文档从平台概念角度说明主要数据对象，帮助开发时判断归属和引用关系。字段细节以 Backend model、schema 和 migration 为准。

## 身份与权限

- 用户：登录主体，拥有平台账号、会话和 AI 配置。
- 工作空间：团队资产和权限隔离单位。
- 成员关系：连接用户与工作空间，决定可见项目、资源和操作权限。

## 创作对象

- 项目：页面集合、主题样式和构建配置的上层容器。
- 页面：可预览、截图和构建的内容单元，保存源码、版本和页面配置。
- 页面版本：记录页面历史内容，用于回看、恢复和审计。

## 资产对象

- 资源：图片、图标、视频、字体、DrawIO、Mermaid、图表、公式等素材。
- 组件：工作空间级 Vue 组件，包含草稿、发布版本和 previewSchema。
- 主题：品牌色、Logo、项目图标、字体绑定等视觉基础配置。
- 样式：页面展示配置、Markdown 样式规范和通用排版约束。

## AI 运行态

AI 会话、run、事件、消息、工具调用和 HITL requirement 的事实源在 Backend 主库 `ai_agent_*` 表中。Redis 不保存 AI run/HITL 事实源，只保存预览、截图、构建等临时运行态。

## 预览与构建对象

- preview artifact：Backend 为 Runtime 预览准备的短期上下文。
- build snapshot：Backend 为一次构建固化的项目快照。
- 构建任务：记录构建状态、日志摘要、产物路径和错误信息。
- 构建产物：Runtime 上传回 Backend 后由 Backend 托管或提供下载。

## 引用关系

资源、组件、主题、样式和页面之间存在引用关系。删除、替换、发布或升级前，应优先检查引用范围；涉及跨模块契约的变更必须补充契约测试。
