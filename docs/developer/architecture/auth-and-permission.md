# 认证与权限

认证与权限由 Backend 统一控制，Editor 和 Runtime 不应绕过 Backend 自行判断用户可见范围。

## 登录与会话

用户通过 Editor 登录，Backend 创建会话并通过 Cookie 维护登录态。本地开发时应避免混用 `localhost` 和 `127.0.0.1`，否则可能因 Cookie 站点不一致导致接口返回 `401`。

## 工作空间隔离

工作空间是用户资产和项目的隔离边界。所有项目、页面、资源、组件、主题、样式和 AI 会话都应在工作空间成员权限下访问。

## Runtime 权限边界

Runtime 不直接面向用户鉴权。Backend 为预览、构建和诊断签发短期令牌，Runtime 校验令牌 audience 和有效期后才能读取上下文或执行任务。

## AI 权限边界

AI 上下文注入和工具调用都必须经过 Backend 权限校验。工具确认只代表用户同意某次操作，不代表绕过用户、工作空间或对象权限。

## 测试要求

认证、权限和工作空间隔离变化时，应补充 Backend API 测试和必要的 E2E smoke。跨模块行为变化时，还要更新契约测试。
